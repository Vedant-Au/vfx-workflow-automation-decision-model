# GitHub setup

## Create and upload

1. On GitHub, select **New repository**.
2. Name it `vfx-workflow-automation-decision-model`, set it to **Public**, and leave the initialisation options unticked.
3. Select **uploading an existing file**.
4. Open the extracted folder in Finder and press **Command + Shift + .** to show `.github` and `.gitignore`.
5. Drag all contents—not the ZIP or outer folder—into GitHub.
6. Confirm the staged files include `.github/workflows/quality.yml`, `.gitignore`, `README.md`, `analysis.py`, `data`, `docs`, `outputs` and `tests`.
7. Commit directly to `main` with `Add VFX workflow automation decision model`.

## Verify and present

1. Confirm the README renders the decision-score and sensitivity charts.
2. Under **Actions**, wait for the `quality` workflow to pass.
3. Add description `Process analysis and MCDA for phased AI scheduling in a VFX production workflow.`
4. Add topics: `business-analysis`, `process-improvement`, `mcda`, `workflow-automation`, `python`, `change-management`.
5. Pin this repository for operations, transformation or consulting applications.
