# Phased implementation roadmap

## Phase 1 - Baseline and govern

- Define scheduling, utilisation, revision and reporting KPIs.
- Assign data owners and standardise status definitions.
- Measure current weekly reporting effort and schedule-change latency.

## Phase 2 - AI-assisted pilot

- Pilot one project or production team.
- Integrate capacity and task status with Flow Production Tracking.
- Retain producer approval for assignments and rescheduling.
- Track deadline reliability, manual hours, workload variance and overrides.

## Phase 3 - Scale or stop

- Scale only if data completeness, adoption and control thresholds are met.
- Automate recurring client reports with an approval gate.
- Evaluate the fully automated architecture after the pilot establishes reliable history and integration patterns.

## Guardrails

- Human approval for material scheduling changes
- Client-data access controls and audit logs
- Override and rollback procedures
- Vendor-exit and data-export plan
- Monthly monitoring for workload fairness and model drift
